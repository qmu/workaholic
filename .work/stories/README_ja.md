# ストーリー

このディレクトリには、各ブランチでの開発者の旅を記録するナラティブドキュメントが含まれています。ストーリーはアーカイブされたチケットを一貫したナラティブに統合し、作業の動機、発生した課題、意思決定の経緯を説明します。

## 目的

ストーリーはレビュアーや将来の開発者に高レベルのコンテキストを提供します。チェンジログは個々の変更をリストし、チケットは実装の詳細を記録しますが、ストーリーは一連の作業の背後にある「なぜ」を説明します。
