# Update PR Title When Updating Existing PR

## Overview

When `/pull-request` updates an existing PR, it should also update the PR title to reflect the iconic change(s). The title should name a single iconic change, or use "etc" suffix when there are multiple significant changes.

## Key Files

- `plugins/core/commands/pull-request.md` - The pull-request command that needs modification

## Implementation Steps

1. In the "If PR already exists" section, add `-f title="..."` to the `gh api` PATCH command to also update the title
2. Add instructions for deriving the title:
   - Parse CHANGELOG entries to identify iconic changes
   - If single change: use that change as title (e.g., "Add dark mode toggle")
   - If multiple changes: use first change + "etc" (e.g., "Add dark mode toggle etc")
3. Update the example `gh api` command:
   ```sh
   gh api repos/{owner}/{repo}/pulls/<number> -X PATCH \
     -f title="<derived-title>" \
     -f body="$(cat <<'EOF'
   ## Summary
   ...
   EOF
   )"
   ```

## Considerations

- Keep title concise (GitHub truncates long titles)
- The "etc" suffix clearly signals multiple changes without listing all
- Title derivation uses same CHANGELOG parsing already done for Summary section
