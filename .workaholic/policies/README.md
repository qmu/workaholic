# Policies

Policy documents describing the repository's practices across 7 domains. Policy documents are generated by the scanner subagent.

- [Test](test.md) - Verification and validation strategy, testing levels, coverage targets
- [Security](security.md) - Authentication, authorization, secrets management, security practices
- [Quality](quality.md) - Code quality standards, linting, formatting, review processes
- [Accessibility](accessibility.md) - Internationalization, localization, content accessibility
- [Observability](observability.md) - Logging, monitoring, metrics, tracing practices
- [Delivery](delivery.md) - CI/CD pipeline, release process, deployment practices
- [Recovery](recovery.md) - Backup, disaster recovery, data restoration practices

For technical specifications, see [../specs/](../specs/README.md).
