# ポリシー

7つのドメインにわたるリポジトリの実践を記述するポリシードキュメント。

- [Test](test_ja.md) - 検証と妥当性確認の戦略、テストレベル、カバレッジ目標
- [Security](security_ja.md) - 資産、脅威モデル、認証/認可、セーフガード
- [Quality](quality_ja.md) - コード品質基準、リンティングルール、レビュープロセス、メトリクス
- [Accessibility](accessibility_ja.md) - コンプライアンス目標、i18nサポート、インクルーシブデザイン
- [Observability](observability_ja.md) - メトリクス、ロギングプラクティス、トレーシング、アラート
- [Delivery](delivery_ja.md) - CI/CDパイプライン、デプロイ戦略、リリースプロセス
- [Recovery](recovery_ja.md) - バックアップスケジュール、保持ポリシー、災害復旧

技術仕様については [../specs/](../specs/README_ja.md) を参照。
