# Release Notes: drive-20260208-131649

## Summary

Comprehensive architectural transformation migrating all analyst subagents to a schema-driven lead architecture, reducing scan agents from 17 to 14.

## Changes

### Added
- `define-lead` skill providing authoritative schema template for lead agent definitions (Role, Responsibility, Goal, Default Policies)
- Automatic patch version bump in `/report` command ensuring every PR merge triggers a release
- Explicit `run_in_background: false` constraint on scan agent invocations to prevent Write/Edit permission loss
- 11 new lead skills: `lead-a11y`, `lead-test`, `lead-security`, `lead-quality`, `lead-observability`, `lead-delivery`, `lead-recovery`, `lead-infra`, `lead-communication`, `lead-db`, `lead-architecture`
- Path-scoped rule `.claude/rules/define-lead.md` for automatic schema enforcement on lead files

### Changed
- Scanner subagent orchestration migrated into `/scan` command for real-time per-agent progress visibility
- All 7 policy analysts replaced with domain-specific leads (a11y-lead, test-lead, security-lead, quality-lead, observability-lead, delivery-lead, recovery-lead)
- 3 viewpoint analysts replaced with domain-specific leads (infra-lead, communication-lead, db-lead)
- 4 viewpoint analysts (application, component, feature, usecase) consolidated into single architecture-lead
- Scan agent count reduced from 17 to 14
- Policy badge system removed; policies now only document implemented and executable practices

### Removed
- Scanner subagent (`plugins/core/agents/scanner.md`)
- `/story` command (deprecated)
- 10 analyst agent files replaced by lead equivalents
- `[Explicit]`/`[Inferred]` badge system from policy documents
- `define-lead` skill directory (moved to `.claude/rules/`)
