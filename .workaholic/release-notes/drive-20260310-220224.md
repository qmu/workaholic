# Release Notes: drive-20260310-220224

## Added

- Deterministic review file convention using per-agent files in `reviews/` subdirectories to prevent concurrent write conflicts in trip workflow
- Phase Gate Policy in trip-protocol enforcing leader-only workflow advancement with intra-phase GATE markers
- Artifact Dependencies section documenting strict Direction-to-Model-to-Design data flow
- E2E Assurance Policy for Planner's Phase 2 testing with tool-agnostic guidance and Playwright as recommended default
- Dev environment readiness validation (`validate-dev-env.sh`) for worktree preparation with concurrent isolation support
- Critical Review Policy requiring substantive analysis and constructive proposals for every review
- Opinion Domain and Review Approach sections for all three agents (non-tech, structural, tech perspectives)
- `/report-trip` command in Trippin plugin for generating journey reports and creating PRs from trip artifacts
- `write-trip-report` skill with agent-based report sections and artifact gathering script
- `/ship-drive` command in Drivin plugin for PR merge, deployment, and verification
- `/ship-trip` command in Trippin plugin for PR merge, worktree cleanup, deployment, and verification
- `ship` skill in Drivin plugin establishing the `cloud.md` deployment convention with shell scripts for pre-check, merge, and cloud.md discovery
- `cleanup-worktree.sh` script for removing trip worktrees and local branches after merge

## Changed

- Trip commit message format from `trip(<agent>): <step>` to `[Agent] Descriptive summary` with mandatory description parameter
- Phase 1 Step 2 from concurrent model/design generation to sequential model-then-design ordering
- Renamed `/report` to `/report-drive` in Drivin plugin with cascading reference updates
- Repository README rewritten to present Workaholic as multi-plugin marketplace with per-plugin sections
- Agent responsibility descriptions updated with opinion-framing (Non-Tech Opinion, Structural Opinion, Tech Opinion)
- Moderation Protocol updated to require perspective synthesis during conflict resolution
- Trip agent colors updated: Planner to red, Architect to green, Constructor to blue
- All three agent definitions updated with Synchronization Rule, Review Output, Opinion Domain, and Review Approach sections

## Notes

- The `/ship-drive` and `/ship-trip` commands use a `cloud.md` convention for deployment -- projects without a `cloud.md` file skip deploy/verify steps
- The `/ship-trip` command reuses Drivin's ship skill scripts via cross-plugin path references
- The `/trip` command still requires `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` environment variable
