# Release Notes: drive-20260311-125319

## Added

- System-wide configuration safety harness (`system-safety` skill) with provisioning repository detection script and prohibited operations blocklist for both drive and trip workflows
- Core plugin (`plugins/core/`) housing shared commands that span both Drivin and Trippin workflows
- Unified `/report` command with context-aware routing based on branch pattern (drive-*, trip/*, worktree fallback)
- Unified `/ship` command with context-aware routing including worktree cleanup for trip context
- Context detection script (`detect-context.sh`) in core plugin for branch pattern analysis
- Worktree discovery script (`list-trip-worktrees.sh`) for finding and selecting active trip worktrees from non-trip branches
- Phase rollback mechanism allowing agents to return from Coding Phase to Planning Phase with two-out-of-three consensus vote
- Rollback proposal artifact format and voting mechanics in trip-protocol
- Planning Phase Focus section in Planner agent with explicit MUST/must-NOT guardrails for Direction artifacts
- Concurrent artifact generation in Planning Phase with structured mutual review session (six reviews per round)
- Concurrent agent launch in Coding Phase with convergence gate before sequential review and testing

## Changed

- Trippin agent personality spectrum: Planner rewritten as business visionary, Constructor as technically accountable agent, Architect as structural bridge between business and technical perspectives
- Trip phase names from "Phase 1" / "Phase 2" to "Planning Phase" / "Coding Phase" across all Trippin plugin files
- Planning Phase workflow from sequential Direction-then-Model-then-Design to concurrent generation with mutual review and convergence
- Coding Phase workflow from sequential test-plan-then-implement-then-review-then-test to concurrent launch with convergence gate
- Artifact Dependencies from strict `Direction -> Model -> Design` ordering to concurrent generation diagram
- Agent Teams instruction block in trip command updated for both concurrent Planning Phase and Coding Phase workflows
- Planner agent prohibited from codebase exploration during Planning Phase (behavioral guardrail)
- Removed `/report-drive` and `/report-trip` commands (replaced by unified `/report` in core)
- Removed `/ship-drive` and `/ship-trip` commands (replaced by unified `/ship` in core)
- Removed Drivin's duplicate ship skill (Trippin's copies are canonical)
- Moved branching skill from Trippin to core plugin

## Notes

- The core plugin is a new third plugin in the marketplace alongside Drivin and Trippin
- The concurrent Planning Phase reverses the model-before-design dependency from the previous branch, justified by each agent having a self-contained domain perspective
- System-safety enforcement is instruction-based (not a technical sandbox); agents respect the blocklist through prompt constraints
- The Planner retains Read/Glob/Grep tools for Coding Phase test execution despite the Planning Phase prohibition on codebase exploration
- Version bumped to v1.0.40
