## What's New

### Trip-Drive Workflow Integration

Trip and drive workflows now work together seamlessly. Trip branches can transition into drive-style iterative development, and all commands are aware of active worktrees.

### Changes

#### Added
- **Worktree detection guards** for `/drive` and `/ticket` commands -- warns users when trip worktrees exist before proceeding
- **Trip-drive cross-command compatibility** -- trip branches support `/ticket` and `/drive` with hybrid context detection (`trip_drive`)
- **Trip plan document** (`plan.md`) for session state persistence and resume with phase/step awareness
- **Trip worktree resume prompt** in `/trip` command -- choose to resume existing sessions or create new ones
- **Reusable strip-frontmatter script** for clean PR body generation

#### Changed
- **Agent quality assurance roles differentiated** -- Planner (E2E testing), Constructor (internal testing), Architect (analytical review) with explicit scope boundaries
- **Archive directory naming sanitized** for trip branches (replaces `/` with `-` for flat paths)
- **Context detection extended** with `trip_drive` hybrid routing in `/report` and `/ship` commands
