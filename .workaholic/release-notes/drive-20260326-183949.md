# Release Notes: drive-20260326-183949

## Added

- **Standards plugin** (`plugins/standards/`): New fourth plugin housing qualitative agents (10 leads, 3 managers, 7 analysts/writers) and 22 skills extracted from drivin, establishing a clean separation between policy/documentation concerns and ticket-driving operations
- **Worktree lifecycle management**: `adopt-worktree.sh`, `eject-worktree.sh`, and `list-all-worktrees.sh` scripts in core branching skill for converting drive branches to/from worktrees
- **Workspace guard**: `check-workspace.sh` script and Step 0 guard in `/report` and `/ship` commands to detect uncommitted changes before proceeding, with user prompt to ignore or stop

## Changed

- **Trippin language enforcement**: Three-layer English-only policy (rule file, agent definitions, team lead instructions) across all trippin plugin components
- **Post-completion agent discipline**: Team lead instructions and trip-protocol now prevent spawning arbitrary agents after trip completion; only designated Planner/Architect/Constructor may be re-invoked
- **Inline story display**: `/report` command now outputs the full story markdown after generation for immediate developer review
- **Trip report format compliance**: Strengthened enforcement in both the report command and write-trip-report skill to treat the template as mandatory
- **Drivin plugin scope**: Reduced to ticket-driving operations only (commands, workflow agents, workflow skills); qualitative components moved to standards
- **Cross-plugin references**: Updated scan command, story-writer, and agent skill preloads from `drivin:` to `standards:` prefix for extracted components
