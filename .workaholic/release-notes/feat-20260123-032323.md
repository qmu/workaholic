# Documentation Experience Improvements

## Summary

Delivered fourteen improvements across documentation workflows including silent formatting, actionable sync-doc-specs command, commit hash tracking, branch story generation, and directory restructuring from `doc/` to `.workaholic/`. Together these changes establish a comprehensive knowledge artifact system that reduces cognitive load at every stage of development.

## Key Changes

- Removed post-edit formatting hooks in favor of one-time formatting before PR creation
- Rewrote `/sync-doc-specs` as a comprehensive, self-contained command with step-by-step instructions
- Added `commit_hash` to documentation frontmatter for AI-aware change tracking
- Renamed "Philosophy" to "Design Policy" and documented the cognitive investment principle
- Added branch story generation with performance metrics and AI coaching to the PR workflow
- Renamed `doc/` to `work/` then to `.workaholic/` for hidden directory conventions
