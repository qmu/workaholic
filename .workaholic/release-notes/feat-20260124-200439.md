# Ticket Metadata and Developer Control Improvements

## Summary

Established tickets as the single source of truth by adding YAML frontmatter metadata, eliminating branch changelogs, and giving developers more control over the drive workflow with approval options and auto-commit for ticket files.

## Key Changes

- Added structured YAML frontmatter to tickets (created_at, author, type, layer, effort, commit_hash, category)
- Eliminated branch changelogs directory in favor of ticket frontmatter
- Renamed `/sync-src-doc` to `/sync-work` to match target directory
- Added "Approve and stop" option to the drive command
- Required developer approval before moving tickets to icebox

## Changes

### Added

- YAML frontmatter metadata to ticket files with structured fields
- "Approve and stop" option to drive approval prompt
- Auto-commit for ticket files on creation
- Invocation of `/sync-work` on release

### Changed

- Renamed `/sync-src-doc` to `/sync-work`
- Changed ticket `date` field to `created_at` with ISO 8601 datetime format
- Added explicit prohibition against autonomous icebox moves

### Removed

- `.workaholic/changelogs/` directory (replaced by ticket frontmatter)
