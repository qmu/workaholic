## Summary

Rewrote the README to establish TiDD (Ticket-Driven Development) as the guiding philosophy, renamed `/report` to `/story`, integrated branch creation into `/ticket`, added a history-discoverer subagent for automated related ticket search, and simplified the drive approval flow.

## Key Changes

- Rewrote README with TiDD philosophy, framing Workaholic as an in-repository development system
- Renamed `/report` to `/story` to better reflect narrative-generation purpose
- Integrated branch creation into `/ticket`, removing standalone `/branch` command
- Added history-discoverer subagent and discover-history skill for automated context search
- Flattened `.workaholic/` documentation into `guides/` and `specs/` directories
- Removed "Needs changes" from drive approval options to simplify the loop
