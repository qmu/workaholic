# Skill Composition and Drive Workflow Decomposition

## Summary

Enabled skill-to-skill composition, decomposed the monolithic drive-workflow into four composable skills, added a driver agent for isolated ticket implementation, moved performance-analyst to parallel execution, and introduced intelligent ticket prioritization in the `/drive` command.

## Key Changes

- Allowed skill-to-skill nesting in architecture policy for better composition
- Split drive-workflow (~200 lines) into four composable skills: request-approval, write-final-report, handle-abandon, format-commit-message
- Added driver agent for isolated ticket implementation context via Task tool
- Moved performance-analyst to Phase 1 parallel execution alongside other agents
- Added intelligent ticket prioritization analyzing type, layer, and effort metadata
- Merged generate-changelog into write-changelog to reduce fragmentation
- Added structured commit message format with UX and architecture sections
