# Fix release note selection, refactor discoverer modes etc

## Summary

Housekeeping and structural improvements across the plugin ecosystem: fixed deterministic release note selection in CI, refactored the discoverer agent from history/source/ticket to history/source/policy modes, removed obsolete skill, and fixed trip worktree artifact generation.

## Key Changes

- Fixed release.yml to select release notes by branch name instead of non-deterministic `ls -t`, and backfilled 18 missing release notes
- Refactored Discoverer agent from history/source/ticket to history/source/policy modes, merging duplicate detection into history
- Fixed trip artifacts landing in the main worktree instead of the trip worktree, with BSD sed compatibility fix

## Changes

### Changed

- Renamed root `sh/` directory to `scripts/` and fixed stale path references in standards plugin gather scripts
- Replaced `ls -t` release note selection in release.yml with merge-commit branch name extraction and lexicographic fallback
- Backfilled 18 missing release notes and regenerated release-notes README index
- Merged Discover Ticket into Discover History mode; added Discover Policy mode for standards-aware discovery
- Expanded `search.sh` to search todo and icebox directories alongside archive
- Simplified `define-lead.md` policy section from fixed four-subsection structure to freeform
- Fixed `init-trip.sh` to accept optional working directory for worktree-aware artifact generation
- Replaced GNU-only `sed` with portable `awk` in `trip-commit.sh` for macOS compatibility

### Removed

- Removed `write-trip-report` skill (89 lines) and `gather-artifacts.sh` script (127 lines)
