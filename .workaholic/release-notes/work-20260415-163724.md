# Consolidate lead domains from 7 to 4

## Summary

Consolidated 7 lead domains into 4 (validity, availability, security, accessibility) by merging overlapping concerns, rewrote all policies in a neutral priority-based tone with explicit trade-offs, and downgraded the work-to-standards dependency from hard to soft reference.

## Key Changes

- Merged lead-quality + lead-db into lead-validity, centered on logical comprehensiveness with new TDD, Functional Programming Style, and Pure/Impure Layer Segregation policies
- Merged lead-delivery + lead-observability into lead-availability, adding CI/CD Automation First and Observable by Design policies
- Renamed lead-ux to lead-accessibility
- Rewrote all policies to state what is prioritized, what it is prioritized over, and what trade-off is accepted
- Changed work plugin dependency on standards from hard to soft reference
