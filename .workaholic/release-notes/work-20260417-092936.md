# Eliminate manager tier, redraw core/work boundary, thin work-side umbrellas

## Summary

Eliminated the manager tier from the standards plugin, redrew the core/work plugin boundary so core becomes a pure reusable-skill library while work holds code-aware orchestration, thinned every work-side command and subagent into a frontmatter-plus-stub alias of its core skill, rewrote the four leading skills in viewpoint-prose with a tech-stack table for Standards, and merged adjacent skills while deleting five dead agents and six orphan skills. Runtime behavior is preserved exactly across every refactor.

## Key Changes

- Removed the entire manager tier (three `manage-*` skills, three `*-manager` agents, `managers-principle`, `/scan` command, `select-scan-agents`, `define-manager` rule, `.workaholic/constraints/`) and wired the four `leading-*` skills directly into work-plugin flows as the project's primary policy lens
- Moved all reusable skills into `plugins/core/skills/` and all code-aware commands/agents/rules into `plugins/work/`, reducing `plugins/standards/` to four `leading-*` skills plus a manifest
- Thinned six work-side umbrellas (drive, report, trip, ticket, ship, discover) so each command and subagent is now an I/O-contract stub that delegates to its core skill
- Renamed `lead-*` to `leading-*` and rewrote Goal/Responsibility as viewpoint prose, Standards as a tech-stack table, and policies in a humble trade-off-acknowledging tone
- Unified `gather-git-context` and `gather-ticket-metadata` into a single `core:gather` skill; absorbed `write-overview` into `core:report`
- Deleted five zero-caller agents (`changelog-writer`, `lead`, `model-analyst`, `terms-writer`, `performance-analyst`) and six exclusively-orphan skills
