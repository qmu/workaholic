# Cross-Agent Distribution and Portable Workflows

## Summary

Workaholic evolved from a Claude-specific monolith into a cross-agent distribution with portable skills installable via the Agent Skills standard. The orchestration was flattened to respect subagent capability boundaries, carry-over concerns were formalized as institutional memory across PRs, and generated artifacts were committed to dist/ and CI-guarded to serve both Codex and the skills CLI from a single source.

## Key Changes

- Established cross-agent skill portability for core and standards plugins via Agent Skills standard (`npx skills add qmu/workaholic`)
- Flattened report, drive, and ticket orchestration from nested subagent chains to direct general-purpose fan-out, deleting per-workflow agent files
- Introduced a carry-over concerns pipeline that propagates branch insights across PR boundaries as first-class institutional memory

## Changes

### Added
- Directory enforcement so ticket-organizer output lands exclusively in .workaholic/tickets/
- Carry-over document corpus under .workaholic/concerns/ for durable branch insights across PRs
- CI-guarded dist/ directory and Dist Freshness workflow to keep generated artifacts in lockstep with plugins/ source
- Codex plugin manifests for cross-agent exposure of portable workflows

### Changed
- Flattened core:report, core:drive, and core:ticket orchestration onto general-purpose subagents, removing per-workflow agent files
- Decoupled core:ship from trip worktree lifecycle for portable distribution
- Reorganized skill dependencies by fan-in to eliminate cross-plugin references
- Made workflow-skill prose agent-neutral for non-Claude agent compatibility

### Removed
- Per-workflow Task-subagent agent .md files (planner, architect, constructor remain as Agent Teams members)
- Inline shell conditionals and text processing from markdown files, extracted to bundled scripts
