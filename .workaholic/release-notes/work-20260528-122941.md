# Hardened Cross-Agent Distribution and Multi-Developer Ergonomics

## Summary

This release syncs plugin versions across Claude and Codex manifests, decomposes the standards plugin into three focused policy-index skills, partitions the ticket queue per developer, and adds a plugin-boundary guard that prevents agents from accessing stale installs or obsolete namespaces.

## Key Changes

- Split the `standards` plugin into three policy-index skills (design / implementation / operation) linking English hard copies of canonical qmu.co.jp articles
- Synced every plugin version through `.claude-plugin/marketplace.json` and added CI validation for Codex manifests to prevent cross-agent version drift
- Partitioned the todo queue into per-user subdirectories (`tickets/todo/<user>/`) with author-routed sweep, preventing one developer's tickets from leaking onto another's branch

## Changes

### Added
- Workflow script smoke tests (hermetic, temp-repo based, exercising branch/context/archive behavior)
- Plugin-boundary guard in all five work commands and canonical `CLAUDE.md` rule
- Per-user subdirectories under `.workaholic/tickets/todo/` with author-routed stray sweep
- CI validation of `.agents/plugins/marketplace.json` and per-plugin Codex manifests

### Changed
- Improved portable workflow skill loadability in generated `dist/workflows/` artifacts via frontmatter rewrite
- Aligned plugin versions across `.claude-plugin/marketplace.json` (single source of truth) and all per-plugin manifests
