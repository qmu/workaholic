# Consolidate plugins into one workaholic plugin and overhaul the ship/release pipeline

## Summary

The three authored plugins (`core`, `standards`, `work`) are now a single `workaholic` plugin, and the ship/release pipeline moved release-note generation and GitHub Release publishing into `/ship`. `/drive` gains an autonomous "night" mode.

## Key Changes

- Merged `core` + `standards` + `work` into one `plugins/workaholic/` plugin (unified manifests, `workaholic:` namespaces, no dependencies)
- Release notes are generated at ship time (not report time), supporting multiple releases per branch
- `/ship` publishes a GitHub Release, deferring to an existing CI release workflow when one is present
- New autonomous `/drive` "night" mode: authorize a batch upfront, run unattended, review a morning report
- Renamed the generated artifacts directory `dist/` → `outputs/`

## Changes

### Added
- Autonomous night-drive mode for `/drive`
- GitHub Release publishing on ship (`publish-release.sh`, CI-aware)

### Changed
- Release-note generation relocated from `/report` to `/ship`
- Three plugins consolidated into one `workaholic` plugin
- Generated artifacts directory renamed `dist/` → `outputs/`
