# Reorder /ship: confirm in production before merging the PR

## Summary

`/ship` now makes the PR merge the final step, gated on a passing production confirmation: it deploys and confirms from the work branch first, records the evidence, and only then merges. An unconfirmable change never reaches `main` — not merging is the rollback.

## Key Changes

- Reordered the Ship Flow so the merge is the last, confirmation-gated step (pre-check → catch up with main → deploy + confirm → record evidence → merge → publish release)
- Added `catchup-main.sh` (sync the branch with main pre-deploy) and `record-evidence.sh` (record non-secret deployment evidence into the story/PR before merge)
- Authored this repo's own `.workaholic/deployments/marketplace.md` contract, so `/ship` on workaholic exercises a real confirm-before-merge path

## Changes
