# Fix the carry-over pipeline and scrub secrets in deployment evidence

## Summary

Fixes two long-standing carry-over pipeline bugs (the verdict applier silently skipped the documented `{"verdicts":…}` object; concern extraction re-emitted duplicates under each PR prefix) and adds a secret-scrubbing guard so deployment evidence cannot leak credentials into a public PR.

## Key Changes

- `apply-carryover-verdicts.sh` now accepts the judge's `{"verdicts":[…]}` object, so resolved concerns are actually archived (bare array still accepted)
- `extract-carryover.sh` deduplicates concerns by canonical identity across PR prefixes, ending the concern-corpus bloat
- `record-evidence.sh` refuses to write deployment evidence containing secret shapes (cloud keys, tokens, auth headers, PEM keys, `token=`/`password=`)

## Changes

### Changed
- Carry-over verdict applier and concern extractor fixed (object input + canonical dedup); `report` skill prose updated to retire the manual workaround

### Added
- Secret-scrubbing guard in `record-evidence.sh`; 8 hermetic smoke tests
