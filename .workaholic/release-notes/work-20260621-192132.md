# Scope /trip Agent Teams members in frontmatter so non-trip workflows never invoke them etc

## Summary

Six workflow and plugin improvements: tighter boundaries for the `/trip` Agent Teams members, a non-blocking `/ship` ticket guard, an autonomous Night Trip mode, feature-branch preservation on merge, a real policy-lens injection test, and a recorded `## Policies` section that `/drive` and `/trip` read before implementing.

## Key Changes

- Scope the `/trip` Agent Teams members in frontmatter so non-trip workflows never invoke them as subagents
- Make the `/ship` ticket guard non-blocking — note queued todo tickets, but never prompt or block the merge
- Add Night Trip: an autonomous, unattended `/trip night` that runs to completion without stopping to ask the developer
- Keep the feature branch on merge (`gh pr merge --delete-branch=false`), and add a real hermetic policy-lens injection test
- Record a mandatory `## Policies` section in tickets that `/drive` and `/trip` read before implementing
