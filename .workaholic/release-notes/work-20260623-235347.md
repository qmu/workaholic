# Unify /trip and /drive on the ticket abstraction etc

## Summary

`/trip` and `/drive` now share one unit of work — the ticket. A trip's design phase decomposes into tickets, its build phase drives them with three-agent QA, and `/trip` is context-aware: over an existing `/ticket` queue it trip-drives the tickets directly (the `ticket → trip` direction).

## Key Changes

- Trip Planning phase emits tickets from the agreed design (Decomposition gate), each with `## Policies` and a Trip Origin link
- Trip Coding phase becomes a per-ticket drive loop with three-agent QA as the gate
- `/report` trip-mode builds Changes from archived tickets and links the trip rationale
- Context-aware `/trip`: a populated todo queue runs queue-execute (skip design, trip-drive)
- The unified "sources × executors" model documented in CLAUDE.md and README

## Changes

### Added
- Trip Planning-phase Decomposition gate (design → tickets)
- Context-aware `/trip` queue-execute mode (trip-drive an existing ticket queue)

### Changed
- Trip Coding phase reworked into a per-ticket drive with three-agent QA
- `/report` trip-mode converges with drive-mode; links the trip rationale
- CLAUDE.md and README document the unified sources × executors model
