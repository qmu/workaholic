# Harden ticket-structure enforcement so /ticket and /drive cannot create non-canonical layouts

## Summary

Closes the structural-enforcement gap in workaholic's ticket management. The only guard policing where tickets live was a PostToolUse `Write|Edit` hook, blind to `bash mv`/`mkdir` — the move path that let a consumer repo accrue an invented `done/` directory, archives nested at `todo/<user>/archive/`, and root-level `todo/` strays. This release adds a complementary shell-move guard and tightens two latent gaps in the existing validator.

## Key Changes

- Tightened `validate-ticket.sh`: `todo/<user>/` is now mandatory (root strays rejected), `abandoned/` is recognized as canonical (fixing a latent rejection bug), and `done/` / nested `todo/<user>/archive/` are rejected
- Added `guard-ticket-structure.sh` — a PreToolUse `Bash` guard (POSIX sh) that blocks `mv`/`cp`/`mkdir`/redirect commands placing a ticket into a non-canonical location, the move path the Write/Edit hook never saw
- Registered the guard in `hooks.json`; added 18 hermetic tests (113 passing)
