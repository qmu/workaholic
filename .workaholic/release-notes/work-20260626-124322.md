# Enforce canonical .workaholic/ layout via allowlist + hook gate etc

## Summary

Makes the canonical `.workaholic/` layout machine-enforced through a single-source allowlist and a guarded Write/Edit hook, adds a read-only doctor to surface pre-existing drift, and gives the `/ship` gate an explicit, recorded accepted-risk bypass for contexts that cannot confirm production.

## Key Changes

- Canonical `.workaholic/` layout enforcement via a single-source allowlist + guarded Write/Edit hook gate (warn-by-default, strict via opt-in)
- Read-only layout doctor audit that classifies pre-existing violations against the same allowlist and suggests remediations without mutating the tree
- Explicit recorded `/ship` override (accepted-risk bypass) for cannot-confirm contexts, preserving the evidence-gated-merge contract

## Changes

### Added

- Layout doctor (`hooks/layout-doctor.sh`): one-shot, read-only audit that walks the top-level `.workaholic/` tree, classifies each entry against the allowlist, and emits structured findings, remediations, and advisories.
- `/ship` accepted-risk bypass: a deliberately-chosen, recorded "merge without production confirmation" option for the cannot-confirm cases; never the default, and never available for a confirmation that ran and failed.

### Changed

- `.workaholic/` layout enforcement: encoded the permitted top-level subdirectories as a single machine-readable allowlist and extended `validate-ticket.sh` to reject (strict) or warn about writes that land outside it.
