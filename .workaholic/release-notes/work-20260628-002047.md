# Add commit/branch policy enforcement gates and /commit etc

## Summary

The plugin's commit-subject and branch-name conventions are now enforced gates rather than prose. An always-on hook guards the agent surface, an opt-in git hook covers the human terminal, and a new `/commit` command provides a policy-conformant path for small non-ticketed changes.

## Key Changes

- Always-on `PreToolUse(Bash)` hooks block direct `git commit`s with off-policy subjects (Conventional/bracket prefixes or over 50 characters) and branch creations that break the `work-YYYYMMDD-HHMMSS` pattern, routing the caller to the sanctioned script.
- An opt-in git-native `commit-msg` hook plus `install-git-hooks.sh` extends the same subject policy to human-terminal commits via `core.hooksPath`, with safe no-clobber handling. Both layers share one `check-subject.sh` rule source so they cannot drift.
- A new `/commit` command wraps `commit.sh` for small non-ticketed changes, deriving a no-prefix subject and the structured commit body.

## Changes

### Added

- Two always-on commit-subject and branch-name `PreToolUse(Bash)` gates, registered in `hooks.json`.
- Opt-in git `commit-msg` subject hook and `install-git-hooks.sh` installer, sharing a `check-subject.sh` validator with the Bash gate.
- `/commit` slash command (sanctioned path for small non-ticketed changes; prefer `/drive` for ticketed work).

### Changed

- `ensure-worktree.sh` now self-validates its branch-name argument against the canonical pattern.

## Notes

- The `Co-Authored-By: Claude` commit trailer is intentionally retained — every gate enforces commit-subject policy only and never strips or blocks a co-author trailer.
- The commit-subject gates cover the agent's Bash surface; install the optional `commit-msg` hook for human-terminal coverage. Both are bypassable with `git commit --no-verify`.
