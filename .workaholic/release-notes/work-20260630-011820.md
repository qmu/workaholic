# Add /catch read-only command for by-developer catch-up reports etc

## Summary

`/catch` is a new read-only command that scans recent repository activity (a two-week window by default), fans out one collector per active developer to summarize their work across commits, tickets, and stories, and synthesizes the overall development direction — then stands ready for follow-up questions.

## Key Changes

- New `/catch` read-only command for by-developer catch-up reports over a customizable time window
- By-developer architecture that fans out one collector per active contributor and synthesizes the cross-developer direction
- Cross-agent portable Agent Skill shipped via the build, extending workflow portability to non-Claude agents

## Changes

### Added
- `/catch` command, the `catch` skill, and the `scan-window.sh` scan script: group commits by author email joined to ticket-author frontmatter, list branch stories, then summarize each developer and synthesize the overall direction. Shipped cross-agent via `build.mjs` `DEFAULT_TARGETS`; stays an Agent Skill with a sequential fallback (no Agent Teams). Nine new smoke-test assertions; 221 passing.
