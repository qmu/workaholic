# Fix branch guard, surface plugin staleness, enrich /catch and /ticket

## Summary

Diagnosed a branch-guard "not firing" report as a stale plugin install (not a defect), fixed the guard over-blocking piped `git branch` list forms, made plugin staleness visible in `check-deps`, enriched `/catch` with per-developer focus and deployments, and added a mandatory quality-gate interrogation to `/ticket`.

## Key Changes

- Diagnosed the branch-guard "not firing" report as a stale pre-1.0.66 install and fixed the guard's over-blocking of piped `git branch` list forms.
- `check-deps` now surfaces the loaded plugin version and PreToolUse guard presence, so a stale or partial install is visible instead of looking like a broken hook.
- `/catch` gained per-developer focus windows (yesterday+today / this week / last week), struggles, per-branch grouping, a guessed generation style, and this-week deployments/releases.
- `/ticket` now interrogates for a quality gate up front; `/drive` surfaces those acceptance criteria at the approval gate.

## Changes

### Added
- `/catch` per-developer focus windows, struggles, branches, generation-style guess, and this-week deployments/releases.
- `/ticket` mandatory `## Quality Gate` interrogation, surfaced at the `/drive` approval and recorded in the commit `Verify:` key.
- `check-deps` stale-install diagnostics (loaded version + guard presence).

### Changed
- `guard-git-branch.sh` no longer over-blocks piped/redirected `git branch` list forms, while still blocking off-policy branch creation.
