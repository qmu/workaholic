# Add /carry and /explain commands, plus prompt project label and deferred-concerns rename

## Summary

Four Claude-only workflow improvements: interactive prompts now show which repository is asking, the "carry-over" concerns vocabulary is retired to "deferred concerns", and two new commands — `/carry` and `/explain` — are added.

## Key Changes

- Every `AskUserQuestion` now carries a `[workaholic]` project-label prefix in its body, so a developer running several sessions across tmux panes can tell which repo a waiting dialog belongs to.
- Renamed the "carry-over" concerns pipeline to "deferred concerns" across prose, four scripts, and test fixtures — a terminology-only change with no behavior difference.
- Added `/carry`: hand off in-progress work to a fresh session by writing a resumption ticket (or trip checkpoint) that a later `/drive` continues, instead of relying on compaction.
- Added `/explain`: answer a question about the repository and export a printer-ready PDF report, rendered from HTML by a real browser, to a chosen directory or the Desktop (falling back to a consent-gated Home write).

## Changes

### Added
- `/carry` command — capture-only session handoff (resumption ticket / trip checkpoint).
- `/explain` command — repository question answered and exported as a browser-printed PDF report.

### Changed
- Interactive prompts surface the repository identity as a `[workaholic]` prefix.
- "carry-over" concept renamed to "deferred concerns" throughout (scripts, prose, tests).
