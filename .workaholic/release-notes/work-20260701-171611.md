# Fetch and scan remote branches in /catch so pushed work is visible

## Summary

`/catch` now reflects what the whole team has pushed. Before scanning, it best-effort-fetches remote-tracking refs and reads remote branches alongside local ones, so a teammate's pushed-but-unpulled work is no longer invisible to the by-developer report.

## Key Changes

- `/catch` runs a best-effort `git fetch` before scanning, so pushed-but-unpulled teammate work becomes visible
- `scan-window.sh` widened from `--branches` to `--branches --remotes`, with a `fetch_ok` flag and graceful degradation (stale-view note) when the remote is unreachable
- Remote branch names normalized by stripping the `origin/` prefix so remote-only branches attribute cleanly to their developer
- Corrected the read-only claim in `catch/SKILL.md` since fetch writes `refs/remotes/*`

## Changes

### Changed
- `/catch` scanner fetches and scans remote-tracking branches, normalizes branch names, and degrades gracefully on fetch failure; hermetic remote fetch+scan tests added (268 passing)
