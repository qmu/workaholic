# Machine-enforce ship catch-up, prompt labels, and /workaholify setup

## Summary

Three prose-only workflow conventions become machine-enforced guards, and a new `/workaholify` command wires a repository to the standards by reference instead of copying rules into its `CLAUDE.md`.

## Key Changes

- `/ship` now mandatorily catches up with `main` before any deploy, and `catchup-main.sh` classifies a conflict as mechanical (version/lockstep manifests + `outputs/`, auto-reconciled) vs content (halt) — a stale-branch deploy can no longer silently revert work or no-op a release
- A blocking `PreToolUse(AskUserQuestion)` hook rejects any prompt whose question body lacks the `[project]` label, so parallel sessions stay identifiable (resolves deferred concerns #67/#69)
- A thin `/workaholify` command refers a repo to the new `workaholify` gateway skill (which links all six policy pillars) rather than duplicating rules into `CLAUDE.md`; it audits `CLAUDE.md` and adds a non-blocking working-directory advisory

## Changes

### Added
- `guard-askuserquestion-label.sh` — blocking prompt-label enforcement hook
- `/workaholify` command + `workaholify` gateway skill + `audit-claude-md.sh` + non-blocking `guard-working-directory.sh`

### Changed
- `/ship` catch-up is mandatory and conflict-classifying; the agent reconciles mechanical conflicts as routine and halts only on content conflicts
