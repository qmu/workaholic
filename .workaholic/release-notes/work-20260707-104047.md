# Surface missions in the /catch report

## Summary

`/catch` now includes a Missions view: each active mission's derived progress, the merged activity under it this window, and — the headline — the unmerged, in-flight work heading toward it that the merge-time mission changelog cannot yet show. Two unrelated hardening fixes ride along.

## Key Changes

- `/catch` gained a top-level Missions section plus a per-developer attribution line, keeping merged progress and unmerged in-flight work distinct
- The `/catch` scanner emits a read-only `missions[]` block (derived progress, merged changelog events, unmerged in-flight tickets) and carries `mission`/`commit_hash` on every ticket
- Cross-skill shell helper references now resolve uniformly across the source tree, raw installed plugins, and the generated workflow bundle
- A regression guard ensures every script a `hooks.json` invokes by path keeps its executable bit

## Changes

### Changed
- Joined missions into the `/catch` scanner and rendered a Missions view in the report — read-only, progress stays derived
- Fixed installed-plugin cross-skill script helper resolution
- Added a hook-executable regression guard to the smoke suite
