# Missions gain an active/archive lifecycle etc

## Summary

Missions now have a full lifecycle: in-progress missions live under `missions/active/`, ended ones move to `missions/archive/` via the new `/mission close` verb, and repositories still on the old flat layout migrate automatically the next time any mission script runs.

## Key Changes

- Missions now have a complete lifecycle with an active/archive structure mirroring the tickets todo/archive convention
- New `/mission close <slug>` verb ends a mission (achieved or abandoned) with automatic status transition and archival
- A living migration converts legacy flat mission layouts transparently on any mission-script touch
- All workflow seams gained area awareness through one centralized slug resolver; `/ticket` now offers only active missions
- Hermetic test coverage for migration scenarios, both-area slug resolution, and close idempotency

## Changes

### Added

- Split missions into active and archive areas, with a shared slug resolver, living migration, and a close.sh end-mission mutator; fixed the missing `missions` entry in the layout allowlist
