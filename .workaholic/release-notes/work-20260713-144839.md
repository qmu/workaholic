# Add mission lens and self-maintaining concern lifecycle

## Summary

Missions and deferred concerns become self-maintaining: a per-developer mission lens re-anchors each session on its assigned roadmap, and deferred concerns gain a stable identity, update-in-place extraction, a collapsing migration, and a `/report` triage step.

## Key Changes

- Missions carry an `assignee`; a new mission lens hook surfaces each developer's assigned mission (progress + next criterion) on every prompt and stop, gated and non-forcing.
- Deferred concerns are keyed on a stable `concern_id` and updated in place — the carried-from clone accumulation is eliminated.
- A living migration collapses legacy concern carry chains into one fresh file per concern (this repo: 65→32).
- `/report` gains a triage step: judge-proposed A+B compound risks, plus idempotent merge and close mutators the developer drives.

## Changes

### Added
- Mission `assignee` field and always-on mission lens hook.
- `/report` concern triage step with compound-merge and close mutators.
- Worktree-creation protocol copies the root `.env` into new worktrees.

### Changed
- Deferred-concern extraction is now identity-keyed update-or-create (fresh, never re-cloned), with a collapsing migration.
