# Summary mode, mission worktrees, and a release-safety scan

## Summary

Three feature groups: a read-only `summary` mode across `/ticket`, `/trip`, and `/mission`; a persistent mission-worktree model where `/mission` scaffolds a dedicated, drive-ready worktree; and a deterministic release-safety scan that warns at `/report` and blocks pre-merge at `/ship` on credentials, oversized changes, and client-work terminology leakage.

## Key Changes

- `summary` mode reports the current developer's assigned work per command, leaving each command's bare-invocation contract unchanged.
- `/mission "<title>"` spins up a persistent `.worktrees/<slug>/` worktree with the mission statement and ordered kickoff tickets; `/ship` preserves and re-branches it; `/mission close` tears it down. Each worktree gets a unique local port base, and missions carry a Playwright-verified quality gate.
- A new `release-scan` skill scans the branch diff for secrets (hard block), oversized/too-many files (overridable), and leaked terminology (confirmable), with a committed `.workaholic/scan-allow` false-positive allowlist.

## Changes

### Added
- `summary` mode for `/ticket`, `/trip`, `/mission`.
- Mission-worktree primitive + `/mission` create-in-worktree with kickoff tickets, per-worktree ports, and a per-mission quality gate.
- Branch-safety scan engine + committed allowlist, wired into `/report` (warn) and `/ship` (block).

### Changed
- `/mission` create now uses a dedicated worktree; `/ship` preserves a mission worktree; `/mission close` removes it; the mission lens is scoped to the current worktree.

### Fixed
- `list-all-worktrees.sh` dropped the last worktree; `.worktrees`/`.env` could be embedded as a gitlink by `git add -A` (now excluded via `.git/info/exclude`).
