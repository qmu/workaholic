# Report whether the ship flow actually pushed etc

## Summary

Fixes four defects that each reported success without proving anything, then re-aims the mission model: a mission now describes the experience it demands and the tickets that reach it, rather than a quality gate fixed before the work exists.

## Key Changes

- The ship flow reports whether it actually pushed, instead of printing success either way
- A triage-minted compound concern derives its own id and inherits provenance, so the next ship updates it instead of cloning it
- A `work-*` branch no longer reports trip mode because an unrelated trip directory exists
- Writing a ticket without a non-empty `## Policies` or `## Quality Gate` is now rejected
- Unassigned missions surface as claimable instead of being hidden from everybody
- A mission describes its demanded experience; the kickoff quality gate is optional and normally empty
- `/mission` interrogates once, then emits the whole drive-ready ticket set
- A mission-authorized queue drives without the per-ticket prompt, and an unqueued problem met mid-run becomes a ticket rather than a stop
- `/mission close` gained `carried`, handing unmet criteria to a successor mission
- Every handoff states where the mission stands and how far a fresh session can proceed

## Changes

### Changed
- Ship, report, drive, carry, and mission flows; no commands added or removed
