# Close silent-success bugs across the workflow scripts etc

## Summary

Nine tickets hardened the workflow machinery against silent-success bugs — cases where a script reported success while quietly doing the wrong thing. Each masked-failure path was converted into a loud failure or made unreachable, and two operator-facing rough edges in `/monitor` and the mission lens were smoothed.

## Key Changes

- Silent-success bugs closed across five core scripts (`/report` artifacts, `archive.sh`, `refresh-index.sh`, `commit.sh`, base-ref resolution) — each now fails loud or reports the truth instead of exiting 0.
- Base resolution moved onto `origin/<default>`, making the stale-local-`main` class of false diffs and false scan blocks structurally unreachable.
- Mission plumbing corrected: worktrees land on the branch they mint, and authorization resolves from the ticket root, not the process cwd.
- `/monitor` leaves now own the whole of a worktree's work (replan application included), making it a genuine non-blocking dispatcher.
- The `mission-lens` hook summarizes an unchanged roster, so a long `/goal` Stop loop no longer buries the developer's own message.

## Changes

### Changed
- `/report` stages intermediate artifacts per-run and fails loud on wrong-shape payloads.
- `archive.sh` reports what its mission mutators did instead of discarding it.
- `refresh-index.sh` preserves hand-written index content and stops indexing untracked dirs.
- `commit.sh` never silently omits a named file from a commit it reports as done.
- Base ref for `/report` and the release scan is taken from `origin`, not a stale local `main`.
- Mission worktrees land on the branch they report; mission resolution follows the ticket.
- `/monitor` fans replan work out to per-worktree leaves; `mission-lens` summarizes on change.
