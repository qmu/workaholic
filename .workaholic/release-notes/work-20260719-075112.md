# /monitor front-loads decisions and runs unattended with honest completion signals etc

## Summary

/monitor now resolves every foreseeable decision in one up-front batch and runs parallel missions unattended, ending with an honest completion signal. Missions whose direction changed get a sanctioned reorganize-and-carry path, /request needs just one confirmation, and prompting follows a codified necessity rule.

## Key Changes

- /monitor redesigned to front-load every blocking decision upfront and run parallel missions unattended with honest completion signals
- Mission reorganize-carry doctrine: handle mid-flight direction changes by replanning to drop moot criteria and merging with successor missions
- Interaction-necessity rule codified: ask only for genuine decisions; act and report as the default
- /request confirmation collapsed to one verbatim confirmation of destination and body
- Opt-in strict cwd guard enforcement mode for operators preferring hard boundaries

## Changes

### Added

- Opt-in `WORKAHOLIC_ENFORCE_CWD` blocking mode for the working-directory guard, keeping the advisory default
- Always-loaded interaction-necessity rule: decide, state the choice, and invite correction unless a genuine fork exists

### Changed

- /monitor front-loads all claims, authorizations, and replan rulings into one pre-flight batch, then runs unattended with an `ok`/`pending` terminal derived from mission state
- Reorganize-and-carry became the encouraged close for missions whose direction changed, surfaced in /monitor reports and /carry handoffs
- /request now confirms destination and verbatim body in a single prompt, and "file" is used only as a noun
