# Introduce strategy layer above missions etc

## Summary

Strategies become the root artifact: long-lived direction with no completion conditions, executed by missions as bounded overnight plans. The overnight loop gains completion reports with reflections, duration prediction versus actual hours, automatic PRs for completed missions, and a commit-throughput KPI.

## Key Changes

- Strategy layer above missions: longevity moves to direction; missions become execution plans
- Automatic PR creation for completed missions in /monitor
- Per-commit size gating in release-scan makes commit count a meaningful KPI
- Duration prediction and actual-hours recording make planning data-driven
- Ask bar raised to decide-and-record: prompts with a recommendable default are decided, not asked

## Changes

### Added

- Strategy artifact and skill (`type: Strategy`, active/archive areas, create/list/read-relation/retire scripts, OKF-indexed)
- Mission duration prediction and idempotent actual-hours accumulation per /monitor run
- /monitor completion report (fixed reason vocabulary) and dated `## Reflection` read back by the next planning
- Commit-count KPI script surfaced in /catch as an Orchestration Throughput block
- Per-commit changed-lines gate in release-scan

### Changed

- Mission creation and replan resolve and stamp the strategy a mission executes
- /monitor auto-creates the PR for each genuinely complete mission; merge stays /ship
- The Recommended-label test lives once in `rules/interaction.md`, referenced by every interrogation surface
- Documentation unified around the four-layer model: strategy → mission → ticket → commit
