# Mission-worktree base resolution now fetches origin and prefers origin/&lt;base&gt; etc

## Summary

Mission worktrees are now cut from the merged tip (`origin/<base>`) instead of a possibly-stale local ref, so a desk whose local `main` trails origin no longer starts work blind to already-merged PRs. A configured-but-unreachable origin fails loud rather than silently seeding from stale state.

## Key Changes

- Mission-worktree base resolution fetches origin and prefers `origin/<base>`, preventing silent work duplication from stale local branches
- New `list-related-prs.sh` surfaces open PRs already touching the same mission slug during `/mission` replan, before duplicate delta tickets are emitted
- A configured-but-unreachable origin now fails loud with a clear diagnostic; offline and local-only repos remain functional via a surfaced local fallback

## Changes

### Changed
- `create-mission-worktree.sh` fetches origin and resolves the mission-worktree base to `origin/<base>` (the merged tip), failing loud on an unreachable configured origin and falling back to the local ref only when no `origin/<base>` exists
- `/mission` replan surfaces sibling PRs touching the same mission slug via the new `list-related-prs.sh`
- Regression tests (`testMissionWorktreeFetchFirst`) cover the fresh-from-origin, unreachable-fails-loud, and no-origin-fallback paths
