## 1. Overview

This branch fixed two long-standing carry-over pipeline bugs that silently lost resolutions and bloated the concern corpus, and added a secret-scrubbing guardrail so deployment evidence cannot leak credentials into a public PR. The carry-over parser now accepts the judge's `{"verdicts":…}` object (not only a bare array), and concern extraction deduplicates by canonical identity across PR prefixes.

**Highlights:**

1. Fixed `apply-carryover-verdicts.sh` to accept the documented `{"verdicts":[…]}` object — resolving the silent-skip that left resolved concerns un-archived (and required a manual bare-array workaround every `/report`)
2. Added canonical-identity dedup to `extract-carryover.sh` so a concern no longer re-emits under each new PR prefix, stopping the corpus ballooning
3. Added a `scan_secrets()` guard to `record-evidence.sh` that refuses deployment evidence containing secret shapes, preventing credential leaks into the public PR

## 2. Motivation

The carry-over pipeline had been working around two root-cause bugs for several ships: `apply-carryover-verdicts.sh` iterated `json.loads` output as an array, so the judge's `{"verdicts":[…]}` object silently skipped every verdict (resolved concerns were never archived), and `extract-carryover.sh` prefixed slugs with the PR number, re-emitting one logical concern as `42-…`/`43-…`/`44-…` until the corpus ballooned from ~6 unique to ~18. Both undermined the traceability the pipeline exists to provide. Separately, the deployment-evidence capture added on the previous branch wrote its inputs into the version-controlled, public PR story with no automatic guard against a credential slipping in — a safety gap worth closing immediately.

The branch fixed the carry-over input-shape and dedup bugs, hardened evidence capture against secret leakage, and shipped both with smoke-test coverage and regenerated cross-agent artifacts.

### 3-1. Scrub secrets in record-evidence.sh before writing evidence ([8b2b58f](https://github.com/qmu/workaholic/commit/8b2b58f))

Added a `scan_secrets()` guard that greps the free-text inputs for structured secret shapes (cloud keys, GitHub/Slack tokens, bearer/basic auth, PEM private keys, `password=`/`token=`/`api_key=` assignments) and refuses to write (`reason: possible_secret`, non-zero exit) rather than leaking a credential into the public story. Deliberately excludes bare hex/base64 so commit hashes and versions are not false positives.

### 3-2. Fix the carry-over pipeline: accept {verdicts} object and dedup ([e390172](https://github.com/qmu/workaholic/commit/e390172))

`apply-carryover-verdicts.sh` now normalizes input shape (dict → `verdicts` key, list → as-is, else empty; non-dict items skipped), so the judge's object is processed and resolved concerns archive. `extract-carryover.sh` adds a `canon()` helper (strips `<pr>-` and `carried-from-pr-<n>-` prefixes) and skips any concern whose canonical identity already exists among active or archived concerns. Updated `report/SKILL.md` Phase 1 to pass the object directly, retiring the workaround.

## 7. Successful Development Patterns

- **Input normalization at the boundary.** Bug 1's fix normalizes the input shape once in `apply-carryover-verdicts.sh` (dict→array) instead of scattering type guards through the consumer, so the rest of the script works with a single known representation.
- **Idempotency through canonical identity.** The `extract-carryover` dedup derives a stable canonical form (stripping PR/carry prefixes) and consults both active and archived concerns, so re-running extraction is a no-op and resolved concerns are never resurrected.
- **A no-op that mimics success is invisible until the differentiating case.** Bug 1 looked fine because every recent run had only `still_active` verdicts (skipping them matched processing them); it only became provable when a `resolved` verdict needed archiving. The new test asserts a `resolved` verdict actually archives.
- **Precision over recall for a secret guard.** The scan omits generic hex/base64 to avoid flagging commit hashes/versions; a guardrail that false-positives on legitimate evidence trains operators to bypass it. The "allows commit-hash/version" test pins this trade-off.
- **Refuse-by-default for public artifacts.** `record-evidence.sh` refuses (rather than silently redacting) on a suspected secret, pushing sanitization to the source so the credential is removed everywhere, not masked in one place.
