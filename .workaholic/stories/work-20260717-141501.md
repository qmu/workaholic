## 1. Overview

This branch hardened the workflow machinery's failure modes: a run of nine bugfix and enhancement tickets that each closed a way a script could report success while silently doing the wrong thing. The through-line is honesty — a base ref that stops measuring against a stale `main`, a commit that refuses to drop a file it claimed to stage, an archive that stops swallowing its mutators' exit codes, a worktree that reports the branch it actually landed on, and a mission resolver that follows the ticket rather than the ambient cwd. It also carried two operator-facing improvements: `/monitor` now fans replan work out to per-worktree leaves, and the mission lens stops burying the developer's own message under a re-injected roster.

**Highlights:**

1. Silent-success bugs closed across five core scripts (`/report` artifacts, `archive.sh`, `refresh-index.sh`, `commit.sh`, base-ref resolution) — each now fails loud or reports the truth instead of exiting 0 on a masked failure.
2. Base resolution moved onto `origin/<default>`, making the stale-local-`main` class of false diffs and false scan blocks structurally unreachable in the multi-worktree desk layout.
3. Mission plumbing corrected end to end: worktrees land on the branch they mint, and mission authorization resolves from the ticket root rather than the process cwd.
4. `/monitor` became a genuine non-blocking dispatcher — leaves own the whole of a worktree's work (replan application included), not just the drive.
5. The `mission-lens` hook now summarizes an unchanged roster to a one-liner, so a long `/goal` Stop loop no longer drowns the developer's prompt.

## 2. Motivation

Most of these defects shared a shape: a script that could finish its job incompletely yet still exit 0 and print a success line, so the failure was invisible until it corrupted something downstream. `/report` read stale or foreign payloads from process-global `/tmp` paths; `archive.sh` discarded every mission mutator's output so a mission that was never rolled passed under "Archive complete!"; `commit.sh` could skip an unstageable named path and commit around it; base resolution measured "what is on this branch" against a local `main` that a sibling worktree keeps structurally stale, narrating already-merged history as new and risking a false, non-overridable secret block that could make a branch permanently unshippable. The branch set out to convert each of these masked-failure paths into a loud one — or, better, to make the bad state unreachable. The `/monitor` and `mission-lens` work rounded it out by fixing two places where the workflow's own orchestration got in the operator's way.

The branch began as refinement of infrastructure its own earlier life had already shipped (that earlier work merged as PR #89), then extended into genuinely new fixes. Five of the nine tickets are new work; the other four are further fixes layered onto that already-shipped infrastructure — none redundant. Midway through the drive the branch was found to be 28 commits behind `main`; it was merged up to date (merge `1b5e82f4`) with two pre-existing semantic conflicts reconciled by hand, and only then did the final operator-facing tickets land.

### 3-1. /report's intermediate artifacts stop sharing fixed paths ([0c9ebc08](https://github.com/qmu/workaholic/commit/0c9ebc08))

`/report` now stages every intermediate artifact under a private per-run `mktemp` directory and `create-or-update.sh` writes the PR body to a private mktemp file with an EXIT trap, so concurrent runs across repos can no longer read a stale or foreign payload. `apply-deferred-concern-verdicts.sh` takes an expected-count and fails loud on a wrong-shape or empty-when-expected payload instead of reporting `still_active:0` at exit 0.

### 3-2. archive.sh reports what the mission mutators did ([a2cc8381](https://github.com/qmu/workaholic/commit/a2cc8381))

`archive.sh` was discarding every mission mutator's stdout, stderr, and exit code, so a mission silently never rolled still passed under "Archive complete!". It now captures each mutator's output and exit and reports at three volumes — loud on failure, terse on success, explicit on a clean no-op with its reason.

### 3-3. refresh-index.sh preserves hand-written index content ([d5b35641](https://github.com/qmu/workaholic/commit/d5b35641))

`refresh-index.sh` fully regenerated each flat-area `index.md` on every `/report`, silently deleting hand-written prose and indexing empty untracked dirs a fresh clone 404s on. Indexes now carry a marked generated region so prose outside it survives verbatim, and the subdir block lists a directory only when git shows a tracked file under it.

### 3-4. commit.sh never silently omits a file ([a694ff08](https://github.com/qmu/workaholic/commit/a694ff08))

The sanctioned commit path had two staging holes: an unstageable named path was skipped with a warning while the commit proceeded (raw exit 0), and with no file args `git add -u` silently dropped untracked files. A named path that cannot be staged is now a fatal error, and every excluded untracked file is listed by name rather than dropped in silence.

### 3-5. Base ref taken from origin, not stale local main ([fd4fe3ad](https://github.com/qmu/workaholic/commit/fd4fe3ad))

`/report` stories and `/ship` scans measured against a local `main` that a sibling worktree keeps structurally stale, narrating already-merged history as this branch's and risking a false, non-overridable secret block. Base resolution now prefers `origin/<default>` via a single no-network resolver — the same ref the PR is diffed against — making the stale-base class unreachable.

### 3-6. Worktree lands on the branch it reports ([6cf6d394](https://github.com/qmu/workaholic/commit/6cf6d394))

`create-mission-worktree.sh` minted a `work-*` branch and reported it, but with no local `main` git's remote-tracking DWIM discarded the `-b`, landed the worktree on `main`, and manufactured a stray local `main` — an exit-0 JSON that lied. It now resolves the base to a concrete SHA before `git worktree add`, reads the worktree's real HEAD back, and reports that observation, failing loudly on any mismatch.

### 3-7. Mission resolution follows the ticket, not the cwd ([9be370c1](https://github.com/qmu/workaholic/commit/9be370c1))

`mission_resolve` answered a bare slug relative to the process cwd, so the same ticket read a different (or wrong same-slug) `mission.md` from a different cwd, silently borrowing the wrong authorization. `resolve.sh` was rebuilt as `(root, arg)` with root derived from the artifact or repo; all eleven call sites pass an explicit root and it returns absolute paths.

### 3-8. /monitor fans out per-worktree work to leaves ([1c088ac4](https://github.com/qmu/workaholic/commit/1c088ac4))

A `/monitor` run did replan bookkeeping serially in the main agent inside each worktree, defeating the non-blocking dispatcher and serializing worktrees that could progress in parallel. Leaves now own the whole of a worktree's work — replan application (delta tickets, body sections, `drive_authorized`, commit) included — while the main agent keeps only the developer prompts a leaf cannot issue and tunes wave-size concurrency for interference and resource load.

### 3-9. mission-lens summarizes the roster on change ([f8bf3361](https://github.com/qmu/workaholic/commit/f8bf3361))

A long `/goal` Stop condition re-fires `mission-lens` on every turn, re-injecting the full roster and burying the developer's own message. The hook now emits the full roster only when it changed since the last turn of the session (per session and event, cksum-compared, fails open); an unchanged turn collapses to a compact count plus the single next action plus a `/mission summary` pointer.

## 4. Outcome

Nine tickets landed, all bugfix or small enhancement, all verified against the full local chain (`build.mjs`, `verify.mjs`, `validate-metadata.mjs`, `posix-lint`, and the workflow smoke suite, which grew from 835 to over 1,100 passing rows across the branch). The net effect is a workflow layer that no longer lies by omission: the five integrity fixes each converted a masked failure into a loud one or made the bad state unreachable, and the mission plumbing now resolves and reports the branch and authorization it actually acted on. The two operator-facing changes make `/monitor` genuinely parallel and keep the mission lens from drowning the developer's prompt. The branch was also brought current with `main` after discovering it was 28 commits behind, so the report is measured against live `origin/main`.

## 6. Concerns

### (carried from PR #88) Compound concern IDs are only collision-checked at mint time

- **Severity:** low
- **Description:** `merge-concerns.sh` refuses a compound-id collision when minting, but hand-authored or hand-edited concern files are never re-checked, so a manually created duplicate id would go unnoticed until it misroutes an update.
- **How to Fix:** Add a duplicate-id warning to `list-active-deferred-concerns.sh`'s identity migration pass, where every file is already read.

### (carried from PR #88) Monitor's contract lives in prose pinned only by sentinels

- **Severity:** low
- **Description:** The monitor behavioral contract (push-decisions, dispatcher, env lifecycle) is orchestration prose pinned by roughly fifteen sentence-level test assertions; a rewording that keeps the meaning but drops a sentinel fails the suite, while one that keeps the sentinel but drifts the meaning passes it.
- **How to Fix:** When editing monitor prose, update the sentinels deliberately in the same change; if the contract grows, promote stable parts into scripts rather than more sentences.

### (carried from PR #88) Monitor's decision loop has no cross-run deferral memory

- **Severity:** moderate
- **Description:** The push-decisions contract asks blockers one at a time until answered or explicitly deferred, but nothing makes a deferral sticky across invocations — a caller-side loop would re-ask the same deferred decisions every cycle.
- **How to Fix:** Record deferred decisions in the run report and have the next invocation re-ask only when the underlying state changed (or after N runs), so deferral is remembered rather than re-litigated.

### (carried from PR #88) Monitor's dev-environment lifecycle has no test coverage

- **Severity:** moderate
- **Description:** `/monitor` boots each driven mission's declared dev environment at dispatch and stops what it started at the terminal state, but boot failure, port-in-use, and teardown paths have no smoke assertions and no live validation — the E2E ran on a project with no declared dev command.
- **How to Fix:** Add hermetic fixtures for boot-with-conflict, no-declaration (skip), and stop-only-what-we-started; exercise once against a real project with a declared dev command.

### refresh-index tracked-directory filter is scoped only to flat-area subdir blocks

- **Severity:** low
- **Description:** The `refresh-index.sh` fix that lists a directory only when git shows a tracked file under it was applied only to the flat-area subdir block (see [d5b35641](https://github.com/qmu/workaholic/commit/d5b35641) in `plugins/workaholic/skills/okf/scripts/refresh-index.sh`); the `trips/` and `missions/` listings are the same defect class but were left unchanged.
- **How to Fix:** Extend the tracked-file guard to the `trips/` and `missions/` listing paths so a fresh clone never 404s on an untracked-only directory they enumerate.

### archive.sh still masks refresh-index's exit code

- **Severity:** low
- **Description:** Even after `archive.sh` was taught to report its mission mutators, its call to `refresh-index.sh` is still wrapped in `>/dev/null 2>&1 || true` (see [d5b35641](https://github.com/qmu/workaholic/commit/d5b35641) in `plugins/workaholic/skills/drive/scripts/archive.sh`), so an index-refresh failure during archiving is still swallowed.
- **How to Fix:** Capture and report the `refresh-index.sh` exit and output at archive time the way the mission mutators now are, rather than discarding them.

### Worktree readback guard is present but untested

- **Severity:** low
- **Description:** `create-mission-worktree.sh`'s step-2 HEAD readback guard exists in code but is not exercised by any test, and the HEAD-disagreement acceptance row is structurally unreachable after the SHA-resolution fix without mocking git (see [6cf6d394](https://github.com/qmu/workaholic/commit/6cf6d394) in `plugins/workaholic/skills/branching/scripts/create-mission-worktree.sh`).
- **How to Fix:** Add a test that mocks a HEAD/branch disagreement so the readback guard's loud-failure path is actually driven.

### commit.sh no-args default (git add -u vs -A) is an unmade judgement call

- **Severity:** low
- **Description:** `commit.sh` now names every excluded untracked file instead of dropping it silently, but the deliberate choice between keeping `git add -u`'s safety and sweeping with `git add -A` was left unmade (see [a694ff08](https://github.com/qmu/workaholic/commit/a694ff08) in `plugins/workaholic/skills/commit/scripts/commit.sh`); this ships the reporting fix, not the behavior change.
- **How to Fix:** If agents keep missing untracked files despite the by-name warning, revisit — either default to `-A` or require an explicit assert-omission flag.

### HQ desk fast-forward workaround retirement is a cross-repo follow-up

- **Severity:** low
- **Description:** The base-ref fix folded in the local-`main`-staleness fact, but the paired step of retiring the fast-forward workaround that lives in a separate strategy repository was deliberately left as a cross-boundary follow-up rather than reached across from here (see [fd4fe3ad](https://github.com/qmu/workaholic/commit/fd4fe3ad) in `plugins/workaholic/skills/ship/scripts/catchup-main.sh`).
- **How to Fix:** File the workaround-retirement through `/request` against the strategy repository so the cross-repo change is made by its owner, not reached across a boundary.

## 7. Successful Development Patterns

- Watch-it-fail-first proved its worth repeatedly: several tickets watched the exact masked-failure path fail (raw exit 0 with a success line) before the fix, then pass — asserting on `git show --stat` / `git ls-files` / exit code rather than on stdout, so the test pins behavior, not wording.
- Making a bad-state class *unreachable* beats defensively checking for it: basing resolution on a remote-tracking ref (`origin/main` has no local upstream to fall behind) removes the stale-base class entirely, leaving the fail-loud staleness check as defensive rather than load-bearing.
- A stale long-lived branch can carry an API refactor (a 1-arg to 2-arg `mission_resolve`) that conflicts with `main`'s new caller without any textual merge conflict — the mismatch surfaces only in the test suite, not in git. Running the full smoke suite immediately after a merge-up is what caught it.
- Remembering the generated-bundle closure prevents CI surprises: `outputs/workflows` bundles `commit.sh` into the create-ticket and drive skill closures, so an edit to an internal, non-exposed skill still needs a `build.mjs` rebuild or Outputs Freshness CI fails.
- Reporting a script's sub-operations at graduated volume (loud on failure, terse on success, explicit on a clean no-op with its reason) turns a silent orchestration seam into an auditable one without drowning the normal path in noise.
