# Use Kebab-Case for Documentation Files

## Overview

Rename all markdown files in `doc/specs/` from UPPER_CASE to kebab-case (e.g., `GETTING_STARTED.md` → `getting-started.md`). Update the documentation rules to specify kebab-case as the standard. README.md files are excluded from this rule.

## Key Files

- `doc/specs/user-guide/GETTING_STARTED.md` → `getting-started.md`
- `doc/specs/user-guide/COMMANDS.md` → `commands.md`
- `doc/specs/user-guide/WORKFLOW.md` → `workflow.md`
- `doc/specs/developer-guide/ARCHITECTURE.md` → `architecture.md`
- `doc/specs/developer-guide/CONTRIBUTING.md` → `contributing.md`
- `doc/README.md` - Update links
- `doc/specs/README.md` - Update links
- `plugins/core/rules/documentation.md` - Add kebab-case naming rule

## Implementation Steps

1. **Rename files in `doc/specs/user-guide/`**:

   - `mv GETTING_STARTED.md getting-started.md`
   - `mv COMMANDS.md commands.md`
   - `mv WORKFLOW.md workflow.md`

2. **Rename files in `doc/specs/developer-guide/`**:

   - `mv ARCHITECTURE.md architecture.md`
   - `mv CONTRIBUTING.md contributing.md`

3. **Update `doc/README.md`**:

   - Update all links to use new kebab-case filenames

4. **Update `doc/specs/README.md`**:

   - Update all links to use new kebab-case filenames

5. **Update `plugins/core/rules/documentation.md`**:

   - Add file naming convention: "Use kebab-case for all markdown files except README.md"
   - Example: `getting-started.md`, `api-reference.md`, `data-model.md`

6. **Update pending tickets** that reference old filenames:
   - Check `doc/tickets/20260123014944-readme-in-specs-subdirs.md` for examples

## Considerations

- README.md is the exception (universal convention)
- Kebab-case is more readable and URL-friendly than UPPER_CASE
- All internal links must be updated to match new filenames
