# Written Language Policy

All trippin plugin output must be written in English. This applies to:

- Commit messages
- PR titles and bodies
- Code and code comments
- Branch names
- Artifact content (directions, models, designs)
- Review files
- Event log entries
- Agent-generated documentation outside `.workaholic/`

The only exception is content written inside the `.workaholic/` directory, which follows the consumer project's CLAUDE.md language setting (English or Japanese). When the consumer's CLAUDE.md specifies Japanese for `.workaholic/`, trip artifacts stored in `.workaholic/.trips/` may use Japanese.
